# Storage System API Documentation

## Core Operations

### Conversation Management

#### Creating Conversations
```python
create_conversation(conversation_id: str, metadata: Dict[str, Any], workspace_path: str) -> bool
```
- Creates a new conversation with unique ID and metadata
- Returns `True` if successful, `False` if conversation already exists
- Metadata can include roles, topics, and other conversation attributes
- Workspace path is required for context isolation

#### Adding Messages
```python
add_message(conversation_id: str, message: Dict[str, Any]) -> bool
```
- Adds a message to an existing conversation
- Message structure:
  - `id`: Unique message identifier
  - `source_role`: Sending role (e.g., "ES", "SET")
  - `target_role`: Receiving role
  - `content`: Message content (automatically encrypted)
  - `urgent`: Boolean flag for priority messages
- Returns `False` if conversation doesn't exist or message is duplicate

#### Retrieving Messages
```python
get_unread_messages(role: str, workspace_path: Optional[str] = None) -> List[Dict[str, Any]]
```
- Retrieves unread messages for a specific role
- Can be filtered by workspace path
- Messages are ordered by urgency and timestamp
- Returns empty list if no unread messages

### Role State Management

#### Updating Role State
```python
update_role_state(role: str, workspace_path: str, state: Dict[str, Any]) -> bool
```
- Maintains role-specific state per workspace
- State can include:
  - Last action performed
  - Active conversations
  - Role-specific settings
- Returns `True` if update successful

## Error Handling

### Common Error Patterns

1. **Resource Not Found**
   - Returns `None` for get operations
   - Returns `False` for update/delete operations
   - Example: Getting non-existent conversation returns `None`

2. **Duplicate Resources**
   - Returns `False` for creation operations
   - Preserves existing data
   - Example: Creating duplicate conversation returns `False`

3. **Database Errors**
   - Logs error details via logging system
   - Returns safe failure value (`False`, `None`, or empty list)
   - Maintains data consistency

4. **Access Control**
   - Validates role permissions before operations
   - Returns `False` if access denied
   - Logs unauthorized access attempts

## Performance Considerations

### Optimized Queries

1. **Role-Based Lookups**
   - Indexed by source_role and target_role
   - Efficient message retrieval by role

2. **Unread Message Queries**
   - Specialized index for unread messages
   - Filtered by target_role and read status

3. **Workspace-Specific Queries**
   - Indexed by workspace_path
   - Efficient conversation filtering by context

4. **Conversation Timeline**
   - Ordered by timestamp
   - Optimized for chronological access

```
[SOURCE_ROLE]: @TARGET_ROLE: [URGENT] Message content
```
Example:

[SET]: @ES: [URGENT] Security vulnerability detected in communication system
```

#### Message Threading/Replies
```
[SOURCE_ROLE]: @TARGET_ROLE: Re: [conversation_id] Message content
```
Example:
```
[CTW]: @SET: Re: storage-docs-123 Documentation draft ready for review

### 2. Best Practices

#### Message Priority Management
- **Use Urgent Flag When**:
  - Security vulnerabilities discovered
  - System-critical errors detected
  - Time-sensitive decisions needed
  - Blocking issues affecting multiple roles

- **Use Normal Priority When**:
  - Regular status updates
  - Non-blocking questions
  - Documentation requests
  - General coordination

#### Role State Management
1. **State Updates**
   ```python
   update_role_state(role, workspace, {
       "last_action": "documentation_review",
       "active_conversations": ["storage-docs-123"],
       "pending_tasks": ["api_documentation"]
   })
   ```

2. **State Retrieval**
   ```python
   current_state = get_role_state(role, workspace)
   if current_state["last_action"] == "documentation_review":
       # Handle documentation workflow
   ```

#### Workspace Context Handling
1. **Context Isolation**
   ```python
   # Create workspace-specific conversation
   create_conversation("storage-docs-123", {
       "workspace": "/project/storage-system",
       "context": "documentation"
   })
   ```

2. **Context Switching**
   ```python
   # Get messages for specific workspace
   messages = get_unread_messages(role, workspace_path="/project/storage-system")
   ```

#### Error Recovery Patterns
1. **Message Delivery Failures**
   ```python
   try:
       send_message(conversation_id, message)
   except MessageDeliveryError:
       # Retry with exponential backoff
       retry_with_backoff(send_message, conversation_id, message)
   ```

2. **State Synchronization**
   ```python
   if not verify_state_consistency(role, workspace):
       # Rebuild state from conversation history
       rebuild_role_state(role, workspace)
