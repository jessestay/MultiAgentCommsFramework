# Archived Role Communication System

## Overview

This directory contains the archived files from our previous approach to implementing a role communication system. These files represent an external system that was designed to facilitate communication between different AI roles.

## Why This Was Archived

We initially explored building an external system for role communication, but later shifted our focus to using Cursor's built-in rules system (`.cursor/rules`) for role communication. This approach proved more efficient and better integrated with our workflow.

The files in this archive represent our earlier work and are preserved for reference purposes.

## Contents

This archive includes:

1. **Core Implementation Files**:
   - Python and JavaScript files implementing the role communication system
   - MCP (Model Context Protocol) servers
   - Role mention systems
   - Message processing utilities

2. **Documentation**:
   - Integration guides
   - Troubleshooting documentation
   - Quick start guides

3. **Configuration Files**:
   - Package configurations
   - Requirements files
   - Server configurations

4. **Scripts**:
   - Installation scripts
   - Test scripts
   - Server management scripts

## Current Approach

Our current approach uses the `.cursor/rules` directory structure to implement role communication directly within Cursor. This provides a more integrated and efficient solution that leverages Cursor's built-in capabilities.

For the current implementation, please refer to the `.cursor/rules` directory.
