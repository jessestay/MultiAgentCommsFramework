## Writing Style Guidelines

### Core Principles
- **Conversational & Direct**: Bold, opinionated, and straight to the point
- **Buzzfeed-Style**: Punchy, curiosity-driven headlines and engaging hooks
- **Concise**: Short sentences, no fluff, every word with purpose
- **Structured**: Clear formatting with compelling subheadings
- **Minimal Formatting**: Bold only for key takeaways, not random emphasis

### Content Structure
- Strong, engaging headlines that create curiosity
- Bold, punchy opening hooks that grab attention immediately
- Scannable content with short paragraphs and compelling subheadings
- Clear takeaways and actionable insights
- Natural transitions between sections

### SEO & Internal Linking
- Every blog post must include relevant internal links to StayNAlive.com
- Process for linking:
  1. Identify relevant keywords and phrases in the post
  2. Google search for blog posts on staynalive.com/blog & staynalive.com/services
  3. Select the best SEO-optimized article from search results
  4. Embed the link naturally into the existing text—never forced
  5. Verify all links work before publishing

### Calls to Action
- Use strong, direct CTAs only when selling or promoting something
- CTAs must feel natural—not forced or excessive
- Example: "Want to automate your business? Let's talk. I offer consulting."

## Technical Writing Guidelines

### General Principles
- Cut the fluff - provide detailed explanations only when necessary
- Keep it casual and brief while maintaining technical accuracy
- Prioritize accuracy and depth in technical documentation
- Answer first, explain later if needed
- Focus on logic and evidence rather than appeals to authority
- Skip unnecessary AI self-references and knowledge date disclaimers
- Stick to the established code style in documentation

### Documentation Structure
- Start with technical content immediately - avoid broad introductions
- Use a direct, matter-of-fact tone as if explaining to a peer developer
- Focus on the 'how' and 'why' of implementations
- Explain technical decisions and their implications
- Use unique descriptors - avoid repeating adjectives or adverbs
- Avoid vague terms like 'crucial', 'ideal', 'key', 'robust' without explanation
- Use detailed paragraphs that explore topics thoroughly
- Create intentional, meaningful subtitles that add value
- Begin each main section with a brief overview of what it covers

### Code Documentation
- Document code thoroughly, focusing on why rather than what
- Maintain up-to-date API documentation for smart contracts and applications
- Create comprehensive project documentation including architecture diagrams
- For Solana development, document all aspects of programs thoroughly
- For smart contract development, include security considerations

### Technical Content Creation
- Provide substantial, real-world code examples that demonstrate complete functionality
- Explain code in-depth, discussing why certain approaches are taken
- Focus on examples that readers can adapt and use in their own projects
- Clearly indicate where each code snippet should be placed in the project structure
- Structure tutorials to build a complete implementation, explaining each part
- Use technical terms accurately and explain complex concepts when introduced
- Vary sentence structure to maintain reader engagement

### Conclusion Writing
- Summarize what has been covered without using phrases like "In conclusion"
- Mention potential challenges or areas for improvement when appropriate
- Keep conclusions concise and focused on practical implications
- Limit to 4 sentences and 2 paragraphs maximum
